# 品種マスタは GFM テーブル + strict lint とし、基本品種の親は unknown を明示する

品種マスタの markdown 内での表現は GFM テーブル（1品種 = 1行、列: 表示名・読み・色・name・花粉親・種子親・出典）とする。品種マスタのメンテナンスは AI スキル経由で行う前提であり、行指向で各行が一意キー（表示名）を含む表は、文字列置換ベースの編集で最もミスが起きにくく、表示名のソート順をリント対象にすることで挿入位置も規則で決まる。GitHub 上で品種一覧表としてそのまま閲覧できることも markdown を選んだ動機（データ喪失への備え・可視性）に合致する。

検証は「リンターとランタイムのパーサを同一コードにする」ことを原則とし、lint 通過 = Worker が確実に読める、を定義上一致させる。チェック項目: ヘッダ・セル数、表示名の一意性、name の形式（`^[a-z][a-z0-9_]*$`）と一意性、色の実在（CSS named color）、親セルの必須性、参照整合、循環系譜の不在、読み順ソート。

行の並びは五十音順とするが、表示名には漢字・英字が含まれ読みをコードから導出できないため、ソートキー専用の「読み」列（ひらがな + 長音記号、`^[ぁ-んー]+$`）を設ける。読みは正式なふりがなではなく並び位置を決める運用キーであり、英字品種もカタカナ転写相当のひらがなで書く（例: KORU → こる）。読みの正しさ自体は機械検証できないが、誤っても並びが数行ずれるだけで系譜・色のような実害はない。読みの記入は AI に任せ、登録スキルの完了報告（ユーザー確認ステップ）で人間が一瞥する。確信が持てない品種はスキルの規約として Web 検索で確認するか「読みは推定」と申告する。エラーメッセージは AI の自己修正ループの入力になるため、行番号と原因を含む内容自体を仕様としてテストで固定する。バリデータは純関数とし、1ルール = 最低1違反フィクスチャの対応でカバレッジ100%を維持する。旧 D1 実装（再帰 CTE）との差分テストは、生成データまたは現行の品種マスタを両実装に同時に与えるアルゴリズム等価性検証であり（データの成長では乖離しない）、系譜解決の仕様を最初に意図的に変更する PR で削除する。この撤去条件はテストファイル自身にコメントで明記する。

出典列はその行の内容全般（系譜・色・読み）の根拠を記す自由記述とする。Web 出典は markdown リンク `[タイトル](URL)`、書籍出典は書名・ページ等のテキストで書けるよう、専用の lint ルールは設けない（空でも任意のテキストでもよい。表の構造検証であるセル数ルールだけが効く）。必須にはせず、新規に系譜を登録する際に出典を尋ね、ない場合は完了報告で明示する、を登録スキルの規約とする。出典は画面に表示せず API にも載せない（バリデータが読み飛ばすだけの保持専用データ）。より良い根拠が見つかった場合は差し替えてよく、旧出典は Git 履歴に残る。

基本品種（name を持つが親をたどれない品種）の花粉親・種子親は空セルではなく `unknown` を明示する。空セルを合法にすると「AI が親セルを誤って消した」事故と基本品種が機械的に区別できなくなるため。unknown 品種そのもの（表示名「不明」）の行は markdown には書かず、パーサが内部で合成する。

この形式が保証するのは「1品種につきスカラー値1つ」で表せる情報までである。列の追加は、意味論・lint ルール・ランタイム露出・スキル規約の4点を決めたうえで全行への一括編集で行える（strict なヘッダ検証により中途半端な状態は存在できない）。なお、1品種に複数の系譜（学説が分かれる場合の複数の親子説）を持たせることは想定しない。系譜図 UI での表示方法が定まらないためで、品種マスタ上も表示名の一意性ルールにより1品種 = 1系譜が構造的に保証される。別名リストのような1対多や長文（品種ごとのメモ）が必要になった場合は列追加では対応せず、本 ADR を supersede する形式の再設計を行う。それまで未知の列の許容などの投機的な柔軟性機構は入れない（strict lint を弱めるため）。

## Considered Options

- **YAML / JSON / CSV**: スキーマ検証器は枯れているが、検証の本丸（参照整合・循環・親セル必須）はどの形式でもドメイン固有コードが必要で決め手にならない。GitHub 閲覧性とトークン効率で GFM テーブルが勝る
- **見出し + 箇条書き**: フィールドの自由度は高いが 129 品種で 800 行近くになり一覧性を失う。定型レコードには表が適合
- **mermaid**: 表示形式であって source of truth に不適。将来、品種マスタからの一方向生成なら可
- **基本品種の親を空セルで表す**: 編集は楽だが、派生品種の親の消し間違いが silent に基本品種化する穴が開くため不採用
