# 枝変わり品種は元品種の花粉親・種子親をそのまま引き継ぐ

枝変わり（芽条変異）で生まれた品種は交配を経ていないため、花粉親・種子親の組で表す系譜（ADR 0001, CONTEXT.md）に本来は載らない。しかし系譜図には載せたいという要求から、枝変わり品種には**元品種と同じ花粉親・種子親**を設定する（例: あかぎがふじの枝変わりなら、あかぎの親をふじと同じ「でりしゃす・国光」にする）。「元品種はふじ」という由来そのものは系譜図には現れず、出典欄にテキストで残す。

## Considered Options

- **(a) 枝変わりは基本品種（両親 unknown）として扱う** — CONTEXT.md の系譜定義（交配の組）と矛盾しないが、たどれる祖先が系譜図に出ず情報量が乏しい
- **(b) 元品種を花粉親・種子親の両方に設定する**（あかぎの両親＝ふじ）— 「由来はふじ」を系譜図に残せるが、系譜図 UI が同一品種の2ノード表示を想定しておらず破綻の恐れがある
- **(c) 元品種の親をそのまま引き継ぐ（採用）** — 既存の2親モデル・系譜図 UI を変えずに祖先をたどれる。代償として「枝変わりである」という関係は系譜図から失われる

## Consequences

- 枝変わり品種とその元品種は、系譜図上では「同じ親を持つ兄弟」のように見える。「あかぎはふじの枝変わり」という関係性は系譜図からは読み取れない。この事実は出典欄に由来コメント（例:「ふじの枝変わり」）として残す
- 元品種の親が後から訂正された場合、枝変わり品種の親も追随して更新する必要がある（データ上は独立した2行のため、機械的な連動はしない）
- この方式は「1品種＝スカラー値1つ」で表せる範囲（ADR 0003）を超えない。枝変わりという関係種別を第3の列として持たせる拡張は行わない
